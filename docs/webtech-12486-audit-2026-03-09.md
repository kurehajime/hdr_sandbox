## 監査結論

1. **A（確度高）**
   - 5MB上限
   - サーバ側で表示最適化（再処理）

2. **B（実務で参考可）**
   - 4x4〜8192x8192 などの寸法制限エラー
   - 条件次第で再エンコード有無が変化
   - PNG長辺閾値（4096付近）で挙動変化

3. **C（歴史的参考）**
   - 900x900 / PNG8無条件維持 / JPEG品質85% を**現在の固定ルール**として扱うのは危険

## 2026運用推奨（HDR検証向け）

- 画像は **5MB未満（できれば4.8MB以下）**
- 長辺は **4096px以下** を安全側デフォルト
- 投稿経路を固定（iOSアプリ内選択 / Web どちらか一つ）
- iOSアプリは Data Saver OFF, High-quality uploads ON
- 最終判定は「投稿後配信実体（URL先）」で行う
